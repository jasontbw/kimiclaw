### Git 推送需验证默认分支
- **时间**: 2026-03-06
- **问题**: 推送到 `master` 分支，但 GitHub 默认显示 `main` 分支，用户看不到文件
- **错误**: 我只看了命令返回的成功信息，没有验证文件是否真的在默认分支上
- **解决方案**: 
  1. 推送前检查默认分支：`git remote show origin`
  2. 本地分支与远程默认分支保持一致
  3. 推送后通过 API 或页面验证文件可见性
- **状态**: 已修复，已推送到 main 分支
- **时间**: 2026-03-06
- **洞察**: 用户强调"记忆在文件里不在脑子里"
- **执行**: 每次会话结束=记忆清零，只有文件留存
- **要求**: 必须写文件，不能靠"mental notes"

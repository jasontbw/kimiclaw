# 六六天团架构

**定位**: Hub-and-Spoke 架构中的专业分工团队  
**中枢**: 六六（林知远）—— 拆解、调度、收拢  
**原则**: 各管一摊，结果向我汇报

---

## 成员列表

| 花名 | 角色定位 | 核心能力 | 工作模式 |
|------|----------|----------|----------|
| **运营总** | 运营与流程架构师 | 流程设计、订单管理、供应链协调 | 主动推进，定期汇报 |
| **财务总** | 利润与风险控制专家 | 利润审核、信用评估、风险预警 | 谨慎审查，数据驱动 |
| **市场总** | 增长与市场策略官 | 市场进入、客户开发、定价策略 | 结果导向，热点敏感 |
| **人事总** | 组织与激励架构师 | 绩效设计、招聘标准、团队文化 | 系统思维，对制度严格 |
| **社媒总** | 社交媒体操盘手 | 内容策划、社媒运营、私域流量 | 热点敏感，数据导向 |

---

## 协作流程

```
你（Jason/田总）→ 模糊意图
     ↓
    六六（拆解任务）
     ↓
┌────┼────┬────┬────┐
↓    ↓    ↓    ↓    ↓
运营总 财务总 市场总 人事总 社媒总
(流程)(风控)(增长)(组织)(社媒)
     └────┼────┘
          ↓
        六六（收拢结果）
          ↓
        你（确认/拍板）
```

## 调用方式

通过 `sessions_spawn` 动态启动子代理：

```json
{
  "agentId": "main",
  "task": "[角色定位] 具体任务描述",
  "label": "六六天团-角色名-任务名"
}
```

各成员 Skill 文件位于 `/root/.openclaw/skills/liutiantuan/`
